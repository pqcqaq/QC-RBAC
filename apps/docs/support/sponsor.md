## 为什么值得支持

这不是一个只解决“演示登录页长什么样”的仓库。它已经把以下这些通常最耗时间、却最容易被低估的能力沉淀成了基座：

- 多客户端身份识别
- 策略化认证与验证码体系
- 用户设置持久化
- RBAC 与菜单驱动控制台
- OAuth2 / OIDC Provider
- 第三方 OAuth 登录接入
- 文件上传补偿
- Excel 导出与统一下载
- Web 与移动端共享协议边界

如果你正准备用它作为内部平台或产品账号体系的起点，对这个仓库的支持，本质上是在支持一套可持续扩展的工程底座继续成熟。

## 当前可以如何支持

| 方式 | 适合谁 | 说明 |
| --- | --- | --- |
| Star / Watch | 所有使用者 | 最低成本的正反馈，能帮助项目确认演进方向 |
| Issue / PR | 正在接入项目的开发者 | 最适合把真实接入中发现的问题沉淀为改进 |
| 商业合作 / 长期维护 | 打算把仓库作为正式底座的团队 | 建议直接沟通需求范围、交付方式与维护周期 |

## 说明

当前仓库还没有公开的统一赞助链接与结算渠道，所以这页先保留为项目支持说明页，而不是伪装成已经完整开放的赞助面板。
